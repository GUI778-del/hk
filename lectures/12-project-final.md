# 第12讲：课程项目与期末展示（Final Project）

讲义目标
- 指导学员完成一个端到端的小项目：从数据准备、训练、评估到部署
- 复习并整合课程所学，培养实验记录与复现习惯
- 制定评审标准与展示要求

项目要求（示例）
- 选题示例：图像分类（自定义小数据集）、简单目标检测、文本情感分类、多标签分类等
- 必要产出：
  - 完整的 notebook（数据/训练/评估）
  - 训练脚本（可在本地复现）
  - 模型导出文件（TorchScript 或 ONNX）
  - 一个简单的推理 API（FastAPI + Docker）
  - 实验报告（包含超参、训练曲线、混淆矩阵、结论）
  - 项目演示视频（可选）

评分标准（示例）
- 技术实现（40%）：模型性能与工程实现（训练/导出/部署）
- 实验质量（20%）：实验设计、对比实验与复现性
- 报告与演示（30%）：报告完整度、沟通表达
- 创新与扩展（10%）：附加改进（模型压缩、监控、自动化等）

提交与演示流程
1. 在指定截止日前提交 Git 仓库链接（包含 README、notebooks、models）
2. 进行 10 分钟演示：问题陈述、方法、结果、演示服务（可选）
3. 教师/助教评分并给出改进建议

示例项目模板（README 的基本模板）
- 项目简介
- 数据来源与处理
- 如何运行（环境、依赖、训练命令、导出、部署）
- 结果（表格/图像）
- 已知问题与未来工作

后续建议
- 鼓励使用小型云实例或学校计算资源完成训练
- 推荐把关键实验写成小型博客或笔记，便于增强学习效果
